# 实时资讯功能使用指南

## 功能概述

新增的实时资讯功能可以帮助你：
- 实时监控竞争对手的社媒动态、官网更新和新闻发布
- 自定义添加和管理资讯源
- 按类型筛选资讯内容
- 自动定时更新最新资讯

## 使用方法

### 1. 查看实时资讯
- 点击导航栏的"实时资讯"标签
- 使用筛选按钮查看不同类型的资讯：
  - 全部：显示所有资讯
  - 社交媒体：Twitter、LinkedIn等社媒平台
  - 官方网站：公司官网、博客等
  - 新闻发布：新闻稿、媒体报道等

### 2. 管理资讯源
- 点击"管理资讯源"按钮打开管理界面
- 添加新资讯源：
  - 输入资讯源名称（如：Tesla官方Twitter）
  - 输入资讯源链接
  - 选择资讯源类型
  - 点击"添加资讯源"
- 删除不需要的资讯源

### 3. 自动更新
- 系统每5分钟自动检查更新
- 最多保存最新20条资讯
- 资讯按时间倒序排列

## 技术实现

### 当前功能
- 使用localStorage保存用户自定义的资讯源
- 模拟数据展示功能效果
- 响应式设计，支持移动端

### 扩展建议
为了获取真实的资讯数据，可以集成以下API：

1. **Twitter API v2**
   - 获取指定用户的最新推文
   - 需要申请开发者账号和API密钥

2. **RSS解析**
   - 使用rss2json.com等服务解析RSS源
   - 适用于大多数官网和新闻网站

3. **新闻API**
   - NewsAPI.org：获取全球新闻
   - Google News API：搜索特定关键词新闻

4. **网页抓取**
   - 使用Puppeteer或类似工具抓取网页内容
   - 需要服务器端支持

## 部署说明

### GitHub Pages部署
1. 将所有文件推送到GitHub仓库
2. 在仓库设置中启用GitHub Pages
3. 选择主分支作为发布源

### 注意事项
- 由于浏览器CORS限制，某些API调用需要服务器端代理
- 建议使用Netlify或Vercel等平台获得更好的API支持
- 考虑使用GitHub Actions定期更新资讯数据
